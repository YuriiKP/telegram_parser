# Telegram Parser Bot

Многофункциональный Telegram бот для парсинга данных из Telegram с системой подписок и оплаты.

## Описание проекта

### Основные функции:
- **Парсинг участников чатов** - сбор информации о пользователях в Telegram группах
- **Парсинг активных пользователей** - анализ пользователей, которые пишут в чатах
- **Парсинг комментаторов каналов** - сбор данных о пользователях, оставляющих комментарии
- **Парсинг подписчиков каналов** - анализ аудитории Telegram каналов
- **Система подписок** - платный доступ к функциям парсинга
- **Интеграция с YooKassa (через телеграм)** - безопасные онлайн-платежи
- **Административная панель** - управление ботом и пользователями
- **Экспорт данных** - выгрузка результатов в удобных форматах

### Технологический стек:
- **Python 3.12+** - основной язык программирования
- **aiogram 3.x** - современный фреймворк для Telegram Bot API
- **SQLAlchemy 2.0** - ORM для работы с базой данных
- **Telethon/opentele-py313** - библиотеки для парсинга Telegram
- **Docker** - контейнеризация приложения

## Архитектура

Проект построен по модульной архитектуре с четким разделением ответственности:

### Структура проекта:
```
telegram_parser/
├── main.py                    # Точка входа приложения
├── loader.py                  # Инициализация зависимостей (бот, БД, конфигурация)
├── storage.py                 # Модели базы данных и управление БД
├── Dockerfile                 # Конфигурация Docker
├── docker-compose.yaml        # Оркестрация контейнеров
├── handlers/                  # Обработчики сообщений
│   ├── user/                 # Пользовательские обработчики
│   └── admin/                # Административные обработчики
├── middlewares/              # Middleware компоненты, проверка подписок пользователей
├── parser/                   # Модули парсинга, основная логика парсинга
├── keyboards/                # Клавиатуры бота
├── utils/                    # Вспомогательные утилиты
├── filters.py                # Фильтры для обработчиков
├── commands.py               # Регистрация команд бота
├── subscription_checker.py   # Фоновая проверка подписок
├── worker.py                 # Воркер для обработки задач парсинга
├── accounts/                 # Хранение сессий аккаунтов для парсинга
└── data/                     # Хранение данных и результатов парсинга
```

### Ключевые компоненты:

1. **База данных (storage.py)**:
   - Модель `User` - информация о пользователях бота
   - Модель `ParsingTask` - задачи парсинга (в разработке)
   - Модель `SystemAccount` - системные аккаунты для парсинга (в разработке)
   - Асинхронное подключение через SQLAlchemy

2. **Система подписок**:
   - Middleware для проверки статуса подписки
   - Фоновая задача для проверки истечения подписок
   - Интеграция с платежной системой YooKassa

3. **Парсинг данных**:
   - Поддержка различных типов парсинга (участники, комментаторы и т.д.)
   - Управление системными аккаунтами
   - Асинхронная обработка задач

4. **Административный функционал**:
   - Управление пользователями
   - Мониторинг задач парсинга
   - Статистика и аналитика

## Быстрый старт

### Предварительные требования:
- Python 3.12 или выше
- Docker и Docker Compose (опционально)
- Telegram Bot Token от [@BotFather](https://t.me/botfather)
- Аккаунт YooKassa для приема платежей (опционально)

### Установка и запуск:

#### Способ 1: Запуск с Docker (рекомендуется)

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd telegram_parser
```

2. Создайте файл конфигурации `.env` на основе примера:
```bash
cp .env.example .env
```

3. Отредактируйте файл `.env`:
```env
TG_TOKEN=your_telegram_bot_token
TG_ADMIN=your_admin_user_id
SQLALCHEMY_DATABASE_URL_TG=sqlite+aiosqlite:///data/bot.db
YOO_KASSA_PROVIDER_TOKEN=your_yookassa_token
```

4. Запустите приложение с Docker Compose:
```bash
docker-compose up -d
```

5. Проверьте логи:
```bash
docker-compose logs -f bot
```

#### Способ 2: Локальная установка

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd telegram_parser
```

2. Создайте виртуальное окружение:
```bash
python -m venv .venv
source .venv/bin/activate  # На Windows: .venv\Scripts\activate
```

3. Установите зависимости:
```bash
pip install -r requirements.txt
```

4. Создайте и настройте файл `.env` (как в способе 1)

5. Создайте необходимые директории:
```bash
mkdir -p data accounts
```

6. Запустите бота:
```bash
python main.py
```

### Настройка базы данных:

По умолчанию используется SQLite. Для использования PostgreSQL измените URL:
```env
SQLALCHEMY_DATABASE_URL_TG=postgresql+asyncpg://user:password@localhost/dbname
```
